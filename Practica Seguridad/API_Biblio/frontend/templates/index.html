{% if api_version == "v1" %}
Modo V1 (Sin Seguridad)
Las peticiones se dirigen a api_v1. No se validan cabeceras, firmas ni identidades. Cualquier cliente con red accesible puede consultar y realizar préstamos.
{% elif api_version == "v2" %}
Modo V2 (API Key)
Las peticiones se dirigen a api_v2. Cada endpoint valida de forma obligatoria que la cabecera x-api-key esté presente y contenga la clave válida configurada en el archivo de entorno.
{% elif api_version == "v3" %}
Modo V3 (API Key + JWT)
Las peticiones se dirigen a api_v3. Además del control por API Key, se requiere iniciar sesión para obtener un token JWT. Cada petición posterior incluye el token en la cabecera Authorization: Bearer para validar la sesión del usuario.
{% elif api_version == "v4" %}
Modo V4 (RBAC & BOLA Protection)
Las peticiones se dirigen a api_v4. Valida API Key, JWT y además implementa Control de Acceso por Roles (RBAC) (ej. solo 'admin' crea libros). Previene vulnerabilidades BOLA/IDOR asegurando que un lector solo cree, devuelva o cancele sus propios préstamos.
{% elif api_version == "v5" %}
Modo V5 (Rate Limiting)
Las peticiones se dirigen a api_v5. Incluye todas las capas de seguridad de V4 y añade Rate Limiting (limitación de peticiones por minuto) para evitar ataques de denegación de servicio (DoS) o fuerza bruta.
{% endif %}
