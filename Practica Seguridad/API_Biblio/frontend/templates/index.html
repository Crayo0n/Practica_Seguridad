Las peticiones se dirigen a api_v2. Cada endpoint valida de forma obligatoria que la cabecera x-api-key esté presente y contenga la clave válida configurada en el archivo de entorno.

Modo V3 (API Key + JWT)
Las peticiones se dirigen a api_v3. Además del control por API Key, se requiere iniciar sesión para obtener un token JWT. Cada petición posterior incluye el token en la cabecera Authorization: Bearer <token> para validar la sesión del usuario.
