Modo V1 - API sin seguridad

Esta versión no requiere API Key, JWT, login ni roles. Cualquier cliente puede consultar alumnos y maestros directamente.

Modo V2 - API Key

Esta versión requiere una API Key. El frontend envía automáticamente el header: 

x-api-key: ABC123

Modo V3 - API Key + JWT

Esta versión requiere API Key y también login con JWT. Primero debes iniciar sesión. Después, el frontend enviará: 

Authorization: Bearer <token>

Modo V4 - API Key + JWT + Roles + BOLA/IDOR

Esta versión valida API Key, JWT, roles y propiedad del recurso. También incluye una ruta vulnerable para demostrar BOLA/IDOR.

Modo V5 - API Key + JWT + Roles + BOLA/IDOR + Rate Limiter

Esta versión valida API Key, JWT, roles y propiedad del recurso. También incluye una ruta vulnerable para demostrar BOLA/IDOR. Limita solicitudes de la API

Consultas generales

En V1 estas consultas deben funcionar sin credenciales.

En V2 estas consultas usan API Key automáticamente desde el frontend.

En V3 necesitas hacer login antes de consultar endpoints protegidos.

En V4 algunas consultas pueden fallar por rol. Por ejemplo, un alumno no debería listar todos los alumnos.

En V5 algunas consultas pueden fallar por rol. Tambien puede fallar por exceso de intentos.

Consultas por ID


En V4 o V5, si haces login como Isabel, consultar alumno ID 1 debe funcionar, pero alumno ID 2 debe fallar en la ruta protegida.

Demostración BOLA/IDOR

Esta prueba aplica en V4 y V5. Haz login como Isabel y consulta el alumno 2 en la ruta vulnerable.

Comparación recomendada:

  1. Login como isabel@email.com.
  2. Consultar alumno por ID 2 en la ruta protegida. Debe fallar.
  3. Probar ruta vulnerable con ID 2. Debe mostrar datos.
